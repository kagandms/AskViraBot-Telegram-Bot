# Security Policy

## Supported Versions

Security fixes are applied to the latest code on `main`.

## Reporting a Vulnerability

Do not disclose security issues in public GitHub issues or pull requests.

Use GitHub private vulnerability reporting when it is enabled for the repository.
If private reporting is unavailable, contact the repository owner through the
contact details listed on the GitHub profile and include:

- a concise description of the issue
- impact and affected surface
- reproduction steps or a proof of concept
- any suggested remediation

Please allow reasonable time for investigation before public disclosure.
